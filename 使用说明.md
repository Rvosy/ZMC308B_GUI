# ZMC 运动控制系统使用说明

## 项目概述

ZMC 运动控制系统是一款基于 .NET Framework 4.8 开发的 Windows 窗体应用程序，用于控制 ZMC 运动控制器。系统支持通过以太网连接控制器，实现多轴精确运动控制。

### 主要功能
- 以太网连接与设备管理
- IP 自动扫描与设备发现
- 四轴运动控制（Z1、Z2、Z3、X）
- 绝对/相对运动模式切换
- 实时位置监控
- 紧急停止控制
- 位置清零与设置

## 系统要求

### 硬件要求
- Windows 操作系统（Windows 7 或更高版本）
- ZMC 运动控制器
- 网络接口卡（RJ45）
- 与控制器通信的网线

### 软件要求
- .NET Framework 4.8 或更高版本
- zauxdll.dll 和 zmotion.dll 驱动文件
- Visual Studio 2019 或更高版本（用于编译开发）

## 安装说明

### 开发环境安装
1. 安装 .NET Framework 4.8 SDK
2. 安装 Visual Studio 2019 或更高版本
3. 克隆或下载项目源代码
4. 确保项目根目录包含以下文件：
   - `zauxdll.dll`
   - `zmotion.dll`
   - `ZMC.sln`
   - `ZMC.csproj`

1. 将 `zauxdll.dll` 和 `zmotion.dll` 复制到应用程序输出目录
2. 确保 .NET Framework 4.8 已安装
3. 运行 `ZMC.exe`

## 使用方法

### 1. 设备连接

#### 扫描网络设备
1. 点击"扫描"按钮
2. 系统将自动搜索网络中的 ZMC 设备
3. 扫描完成后，设备 IP 地址将显示在 IP 下拉框中

#### 连接设备
1. 在 IP 地址框中选择或输入目标设备 IP
2. 点击"连接"按钮
3. 连接成功后，状态栏显示"已连接"及设备 IP
4. 定时器自动启动，实时更新各轴位置信息

#### 断开连接
1. 点击"断开"按钮
2. 系统停止设备连接
3. 定时器停止，状态更新终止

### 2. 运动控制

#### 设置运动参数
在运动参数设置区域配置以下参数：
- **单位（脉冲当量）**：定义移动距离的计量单位
- **速度**：运动轴的最大速度
- **加速度**：运动加速过程的加速度
- **减速度**：运动减速过程的减速度

#### 选择运动模式
- **绝对运动**：移动到指定绝对坐标位置
  - 终点位置标签显示为"终点位置"
  - 输入目标坐标值

- **相对运动**：从当前位置移动指定增量
  - 终点位置标签显示为"移动增量"
  - 输入移动距离

#### 执行运动
1. 设置各轴目标位置（Z1、Z2、Z3、X）
2. 点击"启动"按钮
3. 系统根据设置参数执行运动
4. 状态栏显示"运动中..."

#### 紧急停止
- 点击"停止"按钮
- 所有轴立即停止运动
- 适用于紧急情况或异常处理

### 3. 位置管理

#### 清零（归零到机械原点）
1. 点击"清零"按钮
2. 所有轴移动到 0 位置
3. 系统根据当前运动模式执行：
   - 绝对模式：直接移动到坐标 0
   - 相对模式：反向移动当前距离到 0

#### 设为零点（电子秤去皮功能）
1. 将设备移动到 desired 位置
2. 点击"设为零点"按钮
3. 当前位置被设置为 0 参考点
4. 后续运动基于新的零点计算

## 界面说明

### 连接区域
- **IP 地址框**：输入或选择设备 IP 地址
- **扫描按钮**：扫描网络中的 ZMC 设备
- **连接按钮**：建立与控制器的连接
- **断开按钮**：断开当前连接
- **状态标签**：显示当前连接状态

### 运动参数区域
- **单位输入框**：设置脉冲当量
- **速度输入框**：设置运动速度
- **加速度输入框**：设置运动加速度
- **减速度输入框**：设置运动减速度

### 运动模式区域
- **绝对运动单选框**：选择绝对坐标运动模式
- **相对运动单选框**：选择相对增量运动模式
- **终点位置标签**：根据模式动态显示提示文本

### 轴控制区域
- **Z1 轴**：当前位置显示 + 目标位置输入
- **Z2 轴**：当前位置显示 + 目标位置输入
- **Z3 轴**：当前位置显示 + 目标位置输入
- **X 轴**：当前位置显示 + 目标位置输入

### 操作按钮区域
- **启动按钮**：执行运动控制
- **停止按钮**：紧急停止所有轴
- **清零按钮**：所有轴归零
- **设为零点按钮**：将当前位置设为0

## 技术架构

### 项目结构
```
ZMC/
├── Program.cs              # 程序入口
├── Form1.cs                # 主窗体逻辑
├── Form1.Designer.cs       # 窗体设计器生成代码
├── Zmcdll.cs               # DLL 接口封装
├── ZMC.csproj              # 项目配置文件
├── ZMC.sln                 # 解决方案文件
├── App.config              # 应用程序配置
├── zauxdll.dll             # ZAux 驱动库
└── zmotion.dll             # ZMotion 驱动库
```

### 核心类说明

#### ZmcDll 类
封装 ZMC 控制器底层 DLL 接口，提供：
- 连接管理：`OpenEth()`, `Close()`, `IsConnected`
- 设备扫描：`SearchEthList()`
- 位置控制：`GetDpos()`, `SetDpos()`
- 参数设置：`SetUnits()`, `SetSpeed()`, `SetAccel()`, `SetDecel()`
- 运动控制：`SingleMoveRel()`, `SingleMoveAbs()`
- 紧急停止：`Rapidstop()`

#### Form1 类
主窗体类，实现用户界面交互逻辑：
- 事件处理：按钮点击、单选框切换、定时器事件
- 状态管理：连接状态、运动状态、位置监控
- 参数验证：输入合法性检查、异常处理

### 轴号映射
```
轴号 0 → Z1 轴
轴号 1 → Z2 轴
轴号 2 → Z3 轴
轴号 3 → X  轴
```

## API 接口说明

### 连接管理
```csharp
// 打开以太网连接
int OpenEth(string ip)         // 返回 0 表示成功

// 关闭连接
int Close()                     // 返回 0 表示成功

// 检查连接状态
bool IsConnected                // true 表示已连接
```

### 设备扫描
```csharp
// 搜索网络中的设备
string SearchEthList(uint timeoutMs)  // 返回 IP 列表字符串
```

### 运动参数设置
```csharp
// 设置脉冲当量
int SetUnits(int axis, float units)

// 设置速度
int SetSpeed(int axis, float speed)

// 设置加速度
int SetAccel(int axis, float accel)

// 设置减速度
int SetDecel(int axis, float decel)
```

### 位置控制
```csharp
// 读取当前位置
float GetDpos(int axis)              // 返回位置值

// 设置位置（设为零点）
int SetDpos(int axis, float value)

// 单轴相对运动
int SingleMoveRel(int axis, float distance)

// 单轴绝对运动
int SingleMoveAbs(int axis, float position)
```

### 紧急控制
```csharp
// 急停
int Rapidstop(int mode)     // mode=2 表示立即停止
```

## 故障排除

### 连接问题
- **症状**：连接失败，提示错误码
- **解决方案**：
  1. 检查网络连接是否正常
  2. 确认 IP 地址是否正确
  3. 检查控制器电源是否开启
  4. 查看错误码并查阅 ZMC 手册

### 扫描问题
- **症状**：扫描未发现任何设备
- **解决方案**：
  1. 确保设备与电脑在同一局域网
  2. 检查网线连接是否正常
  3. 增加扫描超时时间
  4. 确认设备网络配置正确

### 运动问题
- **症状**：运动不执行或异常
- **解决方案**：
  1. 检查运动参数是否合理
  2. 确认设备已正确连接
  3. 查看是否有轴超出限位
  4. 检查输入格式是否正确

### 数据读取问题
- **症状**：位置信息更新异常
- **解决方案**：
  1. 确认连接状态正常
  2. 检查定时器是否运行
  3. 查看是否有通信错误
  4. 重新连接设备
